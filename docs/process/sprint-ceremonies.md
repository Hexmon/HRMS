# Sprint Ceremonies

## Sprint Planning

- Confirm sprint goal.
- Confirm capacity: 48 points per 7-day sprint.
- Select P0/P1/P2 work based on release risk.
- Confirm test data and roles.

## Daily Check

- What changed yesterday?
- What is blocked?
- Any env/config risk?
- Any QA evidence needed?

## QA Handoff

- Developer provides changed modules, roles, env, commands run, and known risks.
- QA runs P0 first, then sprint regression.

## Review

- Demo completed stories against acceptance criteria.
- Confirm deployment notes and rollback paths.

## Retrospective

- Identify defects escaped from dev checks.
- Improve story readiness, QA data, and automation candidates.
