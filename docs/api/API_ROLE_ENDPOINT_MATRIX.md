Backend RBAC/ABAC is authoritative. This matrix is a consumer guide, not a bypass contract.

Finance Web is role-aware even though Finance Manager operations remain restricted. Employees use Finance Web for requester self-service at `/finance/my-expenses` and `/finance/requests/new`; those routes consume the Expenses / Requester APIs above, not Finance Manager operation APIs. Assigned managers perform verification work at `/finance/manager`. Legacy approval shell routes are removed from active use.

The local Docker QA auth model uses one seeded email/password login for all modules. `employee_code` is retained only as a DEV fallback for legacy scripts. Roles are security permissions, designation/position is organization context, and department/hierarchy provides ABAC/object scope. Production SSO, MFA, reset provider, and identity lifecycle remain HIR and are not implied by this matrix.
