# API Error And Toast QA Matrix

Status: in progress, versioned in `hrms-client`.

## Shared Rule

Every API-backed write action must await the backend response before showing a success toast. Backend failures must surface the backend `message`, validation `details`, and `request_id` where present through `formatUserFacingError` / `toastApiError` from `src/shared/api/user-facing-errors.ts`.
